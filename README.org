#+title: folgezett.el

[[https://melpa.org/#/folgezett][file:https://melpa.org/packages/folgezett-badge.svg]]

Luhmann-style folgezettel IDs for [[https://www.orgroam.com/][org-roam]].

When you capture a new org-roam note, =folgezett= prompts you to choose where it belongs: start a new chain, branch from an existing note, or continue an existing chain. It generates a branching ID (e.g. =1.1=, =1.1a=, =1.1a2=) and stores it in the node's =FOLGEZETTEL_ID= property, mirroring the slip-box numbering system described by [[https://writing.bobdoto.computer/how-to-use-folgezettel-in-your-zettelkasten-everything-you-need-to-know-to-get-started/][Niklas Luhmann]].

** ID scheme

IDs alternate between numeric and alphabetic segments:

#+begin_example
1.1          (root)
├── 1.1a
│   ├── 1.1a1
│   │   └── 1.1a1a …
│   └── 1.1a2
└── 1.1b
2.1          (another root)
#+end_example

A digit-terminal parent (e.g. =1.1=) appends a letter for its first child (=1.1a=); a letter-terminal parent (e.g. =1.1a=) appends digits (=1.1a1=). Siblings are assigned sequentially: =a→b→c…= or =1→2→3…=.

** Installation

=folgezett= is available from [[https://melpa.org/#/folgezett][MELPA]].

*** package.el

Make sure MELPA is in =package-archives=, then install with =M-x package-install RET folgezett RET=.

Configure it after =org-roam= loads:

#+begin_src emacs-lisp
(with-eval-after-load 'org-roam
  (require 'folgezett)
  (folgezett-setup))
#+end_src

*** use-package

#+begin_src emacs-lisp
(use-package folgezett
  :after org-roam
  :config
  (folgezett-setup))
#+end_src

*** straight.el / Git checkout

If you prefer installing directly from GitHub:

#+begin_src emacs-lisp
(use-package folgezett
  :straight (:host github :repo "landerwells/folgezett.el")
  :after org-roam
  :config
  (folgezett-setup))
#+end_src

For a manual checkout, clone or download =folgezett.el= and add it to your load path:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/folgezett.el/")
(with-eval-after-load 'org-roam
  (require 'folgezett)
  (folgezett-setup))
#+end_src

*** Doom Emacs

If installing from MELPA, add this to =packages.el=:

#+begin_src emacs-lisp
(package! folgezett)
#+end_src

Then configure it in =config.el=:

#+begin_src emacs-lisp
(use-package folgezett
  :after org-roam
  :init
  (setq folgezett-capture-keys '("m")
        folgezett-db-link-parent t
        folgezett-include-id-in-filename t)
  :config
  (folgezett-setup))

(map! :leader
      (:prefix-map ("n z" . "folgezettel")
       :desc "Assign ID"        "a" #'folgezett-assign-id
       :desc "Goto parent"      "p" #'folgezett-goto-parent
       :desc "List children"    "c" #'folgezett-list-children
       :desc "Show tree"        "t" #'folgezett-show-tree
       :desc "Reparent"         "r" #'folgezett-reparent
       :desc "Reparent subtree" "R" #'folgezett-reparent-subtree))
#+end_src

** Quick start

1. Install =folgezett= from MELPA or GitHub.
2. Call =folgezett-setup= after org-roam loads (see above).
3. Capture a new org-roam note with =org-roam-capture=.
4. Choose a new chain, branch from an existing note, or continue an existing chain.
5. The =FOLGEZETTEL_ID= property is written to the note automatically. If =folgezett-include-id-in-filename= is non-nil, the captured file is renamed after capture finalization.

** Commands

| Command                      | Description                                              |
|------------------------------+----------------------------------------------------------|
| =folgezett-assign-id=        | Assign or reassign an ID to the note at point            |
| =folgezett-remove-id=        | Remove the ID and parent properties from the note         |
| =folgezett-reparent=         | Re-parent the current note (updates its ID only)         |
| =folgezett-reparent-subtree= | Re-parent and recursively update all descendant IDs      |
| =folgezett-goto-parent=      | Jump to the parent note                                  |
| =folgezett-list-children=    | Select and visit a direct child of the current note      |
| =folgezett-show-tree=        | Display the full folgezettel hierarchy in a tree buffer   |

** Configuration

All variables belong to the =folgezett= customization group (=M-x customize-group RET folgezett=).

*** =folgezett-capture-keys=

Restrict the placement prompt to specific capture template keys. When =nil= (the default), every new org-roam capture triggers the prompt.

#+begin_src emacs-lisp
(setq folgezett-capture-keys '("z"))
#+end_src

*** =folgezett-db-link-parent=

When non-nil, parent-child relationships are injected into org-roam's database during indexing. This makes folgezettel parents appear in backlinks, the org-roam buffer, and org-roam-ui without adding any visible text to your note files. Set before calling =folgezett-setup=, then run =M-x org-roam-db-sync= to backfill existing notes.

#+begin_src emacs-lisp
(setq folgezett-db-link-parent t)
#+end_src

*** =folgezett-include-id-in-filename=

When non-nil, the folgezettel ID is prepended to the filename on assignment and after new-node capture finalization (e.g. =1.1a2 My Note Title.org=). Set this before calling =folgezett-setup=.

#+begin_src emacs-lisp
(setq folgezett-include-id-in-filename t)
(folgezett-setup)
#+end_src

*** =folgezett-show-id-in-completions=

When non-nil (the default), folgezettel IDs appear in the org-roam completion interface.

#+begin_src emacs-lisp
(setq folgezett-show-id-in-completions nil)
#+end_src

*** =folgezett-export-parent-label=

Label used for the export-only parent backlink inserted by =org-export-before-processing-functions=. Default: ="Previous"=. Set to =nil= to disable export-time parent link injection.

#+begin_src emacs-lisp
(setq folgezett-export-parent-label nil)
#+end_src

*** =folgezett-id-property=

The org property used to store a note's folgezettel ID. Default: ="FOLGEZETTEL_ID"=.

*** =folgezett-parent-property=

The org property used to store the parent note's org-roam node ID. Default: ="FOLGEZETTEL_PARENT_ID"=.
